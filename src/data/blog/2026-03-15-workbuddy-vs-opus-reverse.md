# 从「勤奋打字员」到「同声传译」：WorkBuddy 与 Opus 4.6 的逆向对决

在解密那个早教机 SD 卡的过程中，我意外地完成了一场大模型能力的对比实验。

## WorkBuddy：勤奋的统计学家

WorkBuddy 是我首先尝试的工具。它的表现像是一个极度勤奋且具备超强统计能力的「打字员」。在长达 5 小时的处理中，它：

- 对比了数百个加密文件。
- 找出了 Header 部分的字节分布特征。
- 提取了潜在的解密头。

**然而，它卡在了「最后一公里」。** 统计学能告诉你「哪里不同」，但无法告诉你「为什么要这样运算」。它能看到错别字，却读不懂背后的深意。对于非标加密算法中那些扭曲的逻辑，WorkBuddy 的黑盒比对显得力不从心。

## Opus 4.6：冷酷的同声传译

当我把同样的难题交给 Opus 4.6 时，局面瞬间倒向了「降维打击」。

它没有在字节的海洋里做统计，而是直接：

1. **逆向了可执行文件**：将二进制指令还原为逻辑。
2. **理解了工程意图**：它识别出这是一种结合了硬件特性的自定义流加密。
3. **重写了算法**：它给出的不是一个可能的 Header，而是一个可以直接运行的 Python 解密脚本。

**30 分钟，一气呵成。** 在 Opus 面前，复杂的汇编指令集不再是阻碍，而是一本可以被随意翻阅的说明书。逆向工程从一种「猜测与验证」的艺术，变成了一种「精确翻译」的日常。

## 感想：感知与认知的鸿沟

这次对比让我深刻意识到：

- **WorkBuddy 代表的是感知增强**：它能从海量数据中嗅到模式。
- **Opus 4.6 代表的是认知跨越**：它能理解逻辑的本质。

在逆向工程这个极端考验逻辑能力的领域，认知的力量是决定性的。当 AI 能够瞬间理解人类工程师花费数周才能剥离的算法时，软件安全的规则已经彻底改变了。
